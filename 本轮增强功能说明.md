# 本轮增强功能说明

本轮已在现有磁力搜索项目基础上完成以下扩展：

## 1. 数据库可切换配置

系统现在默认仍使用内置 SQLite，但后台已经新增 **数据库配置** 区域，可切换为指定的 MySQL 数据库。

| 能力 | 说明 |
| --- | --- |
| 默认数据库 | 内置 SQLite，适合单机和快速启动 |
| 可选数据库 | MySQL |
| 保存前校验 | 后台必须先验证连接成功，验证失败不能保存 |
| 运行时切换 | 保存成功后立即切换数据库连接 |
| 安全处理 | 返回后台页面时对密码做隐藏处理 |

## 2. 统一主题色与黑夜 / 白天模式

前台搜索页与后台管理页已经统一接入站点级主题配置。

| 配置项 | 说明 |
| --- | --- |
| 主题色 | 后台集中配置，前后台统一生效 |
| 显示模式 | 支持 dark / light / auto |
| 终端适配 | 兼容手机端与 PC 端响应式展示 |

## 3. 数据库容量上限与自动停机

后台现在支持设置 **数据库大小上限（GB）**。当数据库体积超出阈值时，爬虫会自动停止，避免持续写入导致磁盘占满。

## 4. 保存磁力时的过滤规则

在爬虫配置中新增了保存前过滤逻辑：

| 规则 | 说明 |
| --- | --- |
| 关键词过滤 | 命中关键词的磁力不入库 |
| 最小文件大小过滤 | 小于阈值的资源不入库 |
| 最大文件大小过滤 | 大于阈值的资源不入库 |

## 5. 后台磁力管理增强

后台磁力管理除了原有的搜索、单条删除和勾选批量删除外，还新增 **按规则批量删除**。

| 删除条件 | 说明 |
| --- | --- |
| 指定关键词 | 批量删除命中关键词的资源 |
| 指定时间前 | 删除某个时间点之前的资源 |
| 文件大小条件 | 按最小 / 最大文件大小范围删除 |

## 6. 高 CPU 限制下的吞吐优化

针对“CPU 调整到 100% 但整体占用仍偏低”的问题，本轮已经在爬虫层增加高档位吞吐放大策略：

| 优化点 | 说明 |
| --- | --- |
| 有效工作线程放大 | 在高 CPU 配置下不再简单等于基础线程数 |
| 队列上限放大 | 提高 metadata 等待队列容量 |
| DHT 调度更积极 | 缩短高档位下的调度延迟 |
| metadata 回退更积极 | 提高高负载下的整体处理效率 |

## 7. Docker 与开源文档同步更新

本轮同时更新了 GitHub 友好的说明文件和部署模板：

| 文件 | 说明 |
